# 引き継ぎメモ（新規セッション向け）

_最終更新: 2026-07-05（全13シェル完了 ＋ 1a・1b・1c・1d をゲームとして遊べる化）_

## このリポジトリは何か

Claude Design（claude.ai/design）からエクスポートされた TRPG UI デザインのハンドオフバンドル。
ルート直下に 13 種類の TRPG UI プロトタイプ（`1a`〜`1m` の `.dc.html`）が入っている。
それぞれ「全8画面＋デバイス3切替＋プレイスタイル3切替」の共通構造を持つ、世界観の異なる UI の「器」。

- `support.js` は Claude Design 自身のプレビュー用ランタイム（React ベースの `x-dc` テンプレートエンジン）。**実装には使わない**。

## 確定済みの実装方針（ユーザーに確認済み）

1. **技術スタック**: プレーン HTML/CSS/JS（ビルド不要、1ファイル完結、ダブルクリックで開ける）
2. **インタラクティブ性**: フルシェル実装
   - 8画面のナビゲーション切替
   - デバイス3切替（スマホ 392×828 / タブレット 900×660 / PC 1240×772）
   - プレイスタイル3切替（ソロ選択時は GM 画面をナビから消し、GM 画面表示中なら top にフォールバック）
3. 見た目はプロトタイプと**ピクセル一致**が目標

### 実装方式（旧方針からの変更点 — 重要）

当初は「1件ずつ手作業でテンプレートを JS 文字列に変換」する方針だったが、
13ファイルの構文が完全に統一されている（`sc-if` / `sc-for` / `{{ 単純変数参照 }}` / `onClick` のみ、
ロジックは `this.state` / `this.setState` のみ）ことを確認したうえで、
**共通ミニランタイムによる一括生成方式**に切り替えた（手転記ミスの排除と原本忠実性のため）:

- `tools/runtime.js` … `x-dc` テンプレートを解釈する約90行のランタイム（`DCLogic` 互換、React 不使用）。
  SVG 名前空間対応（1j に心電図 SVG あり）。状態変更ごとに全再構築（プロトタイプの再レンダーと同等）。
- `tools/build.js` … 各 `.dc.html` から `<helmet>`・テンプレート・ロジックを**原文のまま**抽出し、
  ランタイムを埋め込んだ自己完結 HTML を `implementation/` に一括生成。一覧 `index.html` も生成。
- 再生成: `node tools/build.js`（デザイン原本を直したらこれを再実行するだけ）

**注意**: 生成物の `<template>` 内には `sc-if` / `{{ }}` が意図的に残る（ランタイムが解釈する）。
旧メモにあった「sc-if が残っていないか grep 確認」は本方式では不要・非該当。
`window.__dcApp` にコンポーネントを公開している（動作検証・今後の拡張用）。

### アーキテクチャ

- `tools/engine.js` … 全作品共通。ダイス / d20判定（20=大成功・1=大失敗）/ localStorage 自動セーブ /
  `GameEngine.attach(app, game)` で `renderVals` にゲーム値を合流・`setState` に永続化を注入。
  `app._resetGame()` でゲーム状態のみ初期化（シェル状態は保持）
- `games/<id>.patches.js` … **ビルド時**にテンプレートの静的ブロックを動的テンプレートへ差し替えるパッチ。
  `{ find, replace }`（完全一致・1回のみ）と `{ block: 'isDice', replace }`（sc-if ブロック丸ごと）の2種。
  見た目のスタイル文字列は原本からそのまま転記する（＝デザイン維持の要）。
  ※ block 置換は「元ブロックに入れ子 sc-if が無い」ことが前提（build.js が検査して違反なら失敗する）
- `games/<id>.game.js` … **実行時**。`window.__GAME__ = { id, initialState, persist, migrate, vals }` を定義。
  `vals(app)` が画面用の値とクリック用クロージャ（シェルの `go` と同じ流儀）を返す
- build.js が `games/1x.patches.js` / `games/1x.game.js` の存在を見て自動で取り込む。一覧ページに「遊べる」バッジ
